# 部署运行

## 1. 环境要求

- JDK 8 或以上
- Maven
- MySQL
- Redis
- Kafka
- Elasticsearch
- Python 3.9 或以上

## 2. 配置环境变量

复制 `.env.example`，根据本机环境填写配置。

常见配置包括：

- MySQL 地址、用户名、密码
- Redis 地址和密码
- Kafka 地址
- Elasticsearch 地址
- 图片上传目录

不要把真实密码提交到 GitHub。

## 3. 启动依赖服务

如果使用 Docker Compose：

```bash
docker compose up -d
```

如果手动启动，需要保证以下服务可访问：

- MySQL: `3306`
- Redis: `6379`
- Kafka: `9092`
- Elasticsearch: `9200`

## 4. 初始化数据库

在 MySQL 中执行：

```text
src/main/resources/db/hmdp.sql
```

确认用户、商户、优惠券、笔记等表已经创建。

## 5. 启动 Java 后端

```bash
mvn spring-boot:run
```

默认端口一般为 `8081`，具体以 `application.yaml` 为准。

启动后可以测试：

```http
GET http://localhost:8081/shop-type/list
```

## 6. 初始化 Elasticsearch

先启动后端，然后调用：

```http
POST http://localhost:8081/shop/es/sync
```

再测试搜索：

```http
GET http://localhost:8081/shop/search?keyword=火锅
```

## 7. 启动 Agent 服务

```bash
cd agent-service
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

测试：

```http
GET http://localhost:8000/health
```

## 8. 常见问题

### Redis 连接失败

检查 Redis 是否启动，地址、端口、密码是否和 `application.yaml` 一致。

### Kafka 消费不到消息

检查 topic 是否存在、Kafka 地址是否正确、消费者组是否正常启动。

### Elasticsearch 搜索为空

先确认 ES 服务已启动，再调用 `/shop/es/sync` 同步索引。

### 图片不显示

检查图片上传目录和 Nginx 静态资源目录是否一致。

### 中文文档乱码

文档需要保存为 UTF-8。不要用 PowerShell 的旧编码方式重写中文 Markdown 文件。
