# 代码详解

本文档按核心链路解释代码，方便复习和面试讲解。

## 1. 登录拦截器

相关文件：

- `src/main/java/com/hmdp/config/MvcConfig.java`
- `src/main/java/com/hmdp/utils/RefreshTokenInterceptor.java`
- `src/main/java/com/hmdp/utils/LoginInterceptor.java`
- `src/main/java/com/hmdp/utils/UserHolder.java`

请求进入后先经过刷新 token 拦截器：

1. 从请求头读取 token。
2. 拼接 Redis key。
3. 查询 Redis Hash 中的用户信息。
4. 如果用户存在，转换成 `UserDTO`。
5. 保存到 `UserHolder` 的 `ThreadLocal`。
6. 刷新 token 过期时间。

需要登录的接口再经过登录拦截器。如果 `UserHolder.getUser()` 为空，就说明当前请求没有登录。

## 2. 缓存工具类 CacheClient

相关文件：

- `src/main/java/com/hmdp/utils/CacheClient.java`
- `src/main/java/com/hmdp/utils/RedisData.java`

`CacheClient` 把常见缓存逻辑抽成通用方法：

- `set`：普通写缓存。
- `setWithLogicalExpire`：写入逻辑过期缓存。
- `queryWithPassThrough`：缓存穿透处理。
- `queryWithMutex`：互斥锁重建缓存。
- `queryWithLogicalExpire`：逻辑过期重建热点缓存。

泛型方法的作用是让同一套缓存逻辑复用到不同实体上，例如 `Shop`、`ShopType`。

## 3. 商户详情缓存

相关文件：

- `src/main/java/com/hmdp/service/impl/ShopServiceImpl.java`

商户详情查询流程：

1. 根据商户 id 拼接 Redis key。
2. 查询 Redis。
3. 如果 Redis 有正常 JSON，直接反序列化返回。
4. 如果 Redis 有空字符串，说明数据库曾经查不到，直接返回空。
5. 如果 Redis 没有数据，查询 MySQL。
6. MySQL 查不到时，把空字符串写入 Redis。
7. MySQL 查到时，把商户对象转成 JSON 写入 Redis。

这就是缓存空值方案，主要用于缓解缓存穿透。

## 4. 秒杀 Lua 脚本

相关文件：

- `src/main/resources/sekill.lua`
- `src/main/java/com/hmdp/service/impl/VoucherOrderServiceImpl.java`

Lua 脚本在 Redis 服务端一次性完成：

1. 判断库存是否大于 0。
2. 判断用户是否已经下单。
3. 扣减 Redis 库存。
4. 把用户 id 加入已下单 Set。
5. 返回业务结果码。

Lua 的关键点是原子性。多个 Redis 命令在脚本里一次执行，中间不会被其他请求插入。

## 5. Kafka 异步下单

相关文件：

- `src/main/java/com/hmdp/dto/VoucherOrderMessage.java`
- `src/main/java/com/hmdp/utils/KafkaConstants.java`
- `src/main/java/com/hmdp/service/impl/VoucherOrderServiceImpl.java`

秒杀接口成功后不直接写数据库，而是发送订单消息到 Kafka：

1. 接口线程执行 Lua，快速判断是否能下单。
2. 生成订单 id。
3. 发送 `VoucherOrderMessage`。
4. 立即返回订单 id。
5. Kafka 消费者异步创建订单。

这样可以把高并发请求和数据库写入解耦，起到削峰作用。

## 6. 一人一单

一人一单有三层保障：

1. Redis Lua 前置判断，减少无效请求。
2. Redisson 按用户 id 加锁，避免同一用户并发创建订单。
3. 数据库创建订单前再次查询，作为最终兜底。

如果业务要求更严格，可以给订单表增加 `user_id + voucher_id` 唯一索引。

## 7. Feed 流

相关文件：

- `src/main/java/com/hmdp/service/impl/BlogServiceImpl.java`

用户发布笔记后，系统查询粉丝列表，把笔记 id 写入每个粉丝的 Redis ZSet 收件箱。

查询关注 Feed 时使用 `ZREVRANGEBYSCORE`：

- score 是时间戳。
- 从大到小查询。
- 用 `minTime` 和 `offset` 处理滚动分页。

这种方案适合关注关系不太极端的场景，读取速度快。

## 8. Agent 工具接口

相关文件：

- `src/main/java/com/hmdp/controller/AgentToolController.java`
- `agent-service/main.py`

Java 后端把业务能力封装为工具接口。FastAPI 服务收到自然语言需求后，调用这些工具接口，组合成推荐结果。

这部分的面试重点是：Agent 不直接访问数据库，而是通过后端提供的稳定工具接口完成服务编排。
