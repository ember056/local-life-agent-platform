# 接口文档

本文档只列出项目中的核心接口，方便测试和面试展示。

## 用户接口

### 发送验证码

```http
POST /user/code
```

参数：

- `phone`：手机号

### 登录

```http
POST /user/login
```

请求体：

```json
{
  "phone": "13800000000",
  "code": "123456"
}
```

返回 token。后续请求需要在请求头中携带 `authorization`。

## 商户接口

### 查询商户详情

```http
GET /shop/{id}
```

说明：优先查 Redis，未命中时查 MySQL 并写回缓存。

### 搜索商户

```http
GET /shop/search?keyword=火锅&page=1&size=10
```

说明：走 Elasticsearch 商户索引。

### 同步商户到 ES

```http
POST /shop/es/sync
```

说明：把 MySQL 中的商户数据同步到 Elasticsearch。

## 优惠券接口

### 查询店铺优惠券

```http
GET /voucher/list/{shopId}
```

### 秒杀优惠券

```http
POST /voucher-order/seckill/{id}
```

说明：

1. Redis Lua 判断库存和一人一单。
2. 成功后发送 Kafka 消息。
3. 消费者异步创建订单。

```http

```

```http
GET /follow/common/{id}
```

```http

```

## Agent 工具接口

这些接口由 Java 后端提供，供 FastAPI Agent 编排服务调用。

### 商户搜索工具

```http
GET /agent/tools/shops/search?keyword=火锅&typeId=1&minScore=85&current=1
```

### 商户详情工具

```http
GET /agent/tools/shops/detail?id=1
```

### 店铺优惠券工具

```http
GET /agent/tools/vouchers?shopId=1
```

### 热门笔记工具

```http
GET /agent/tools/blogs/hot?current=1
```

## FastAPI Mock Agent 接口

### 推荐编排

```http
POST http://localhost:8000/agent/recommend
```

请求体：

```json
{
  "query": "找评分高有优惠券的火锅店",
  "type_id": 1,
  "min_score": 85
}
```

说明：

- 默认开启 Mock 模式，不依赖真实大模型，也不强依赖 Java 服务在线。
- Mock Agent 会按“关键词提取 -> 商户搜索 -> 商户详情 -> 优惠券查询 -> 热门笔记查询”的顺序模拟工具编排。
- 如果设置 `AGENT_MOCK_ENABLED=false`，服务会优先调用 Java 后端工具接口，失败时仍会使用 mock 数据兜底。
- 返回结果包含 `recommendations`、`hot_blogs`、`reason` 和 `tool_trace`，方便面试展示工具调用过程。
